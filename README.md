# 📄 Document RAG API: Modern RAG-Powered Document Q&A

Document RAG API is a high-performance, asynchronous RAG (Retrieval-Augmented Generation) pipeline built with **FastAPI** and **Python**. It allows users to upload documents (PDF, DOCX, TXT) and ask complex questions, receiving context-aware answers generated by state-of-the-art LLMs.

---

## ✨ Key Features

- 🚀 **High-Performance API**: Built with FastAPI for speed and asynchronous efficiency.
- 🧠 **Advanced RAG Engine**: Optimized document chunking and vector retrieval.
- 📁 **Multi-Format Support**: Seamlessly handle `.pdf`, `.docx`, and `.txt` files.
- ⚡ **Async Querying**: Built-in support for asynchronous operations to handle multiple requests.
- 🛠️ **Modular Architecture**: Clean separation between the API layer and the RAG logic.
- 📂 **Session Management**: (Coming Soon) Intelligent session handling for context preservation.

---

## 🛠️ Technology Stack

- **Backend**: FastAPI, Uvicorn
- **AI/ML**: RAG Pipeline, Vector Stores, Embeddings

## 🚀 Getting Started

### 📋 Prerequisites
- Python 3.9+
- `pip` or `conda`

### ⚙️ Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Neeraj-0217/Document-RAG-API.git
   cd Document-RAG-API
   ```

2. **Set up virtual environment:**
   ```bash
   python -m venv .venv
   # Windows:
   .venv\Scripts\activate
   # Linux/macOS:
   source .venv/bin/activate
   ```

3. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure Environment:**
   Create a `.env` file in the root directory (refer to `.env.example`).

---

## 🏃 Running the Application

Start the FastAPI dev server:

```bash
uvicorn backend_api.app.main:app --reload
```

The API will be available at `http://127.0.0.1:8000`.  
Check the Interactive Docs: `http://127.0.0.1:8000/docs`.
